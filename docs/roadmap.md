# Spring + External Tomcat Roadmap

목표를 아주 잘 잡았다. `스프링 + 외장 톰캣`으로 가면 스프링 부트가 숨겨주던 경계가 드러나서, 실제 배포 구조를 이해하는 데 훨씬 좋다.

추천하는 공부 순서는 이렇다.

## 1. 구조를 분리해서 이해하기

- 스프링: 비즈니스 로직, DI, MVC, 빈 생명주기
- 서블릿 컨테이너 톰캣: HTTP 요청 수신, 서블릿 실행, 스레드 관리, 세션 관리, WAR 배포

핵심 질문:

1. 요청은 톰캣에 어떻게 들어오는가?
2. 톰캣은 어떤 서블릿을 호출하는가?
3. 스프링 MVC의 `DispatcherServlet`은 톰캣 안에서 어떤 위치인가?

## 2. 가장 작은 WAR 프로젝트를 직접 배포하기

- `jar`와 `war` 차이 보기
- `SpringBootServletInitializer` 역할 확인
- 외장 톰캣 `webapps`에 배포
- 애플리케이션 기동 로그 읽기

## 3. 요청 흐름을 그림으로 정리하기

- 브라우저 요청
- Tomcat Connector
- Servlet Container
- Filter
- DispatcherServlet
- HandlerMapping
- Controller
- Service
- Repository
- View/Response

핵심은 이 흐름을 "코드"와 "로그"로 같이 확인하는 것이다.

## 4. 톰캣 개념을 따로 떼서 공부하기

꼭 알아야 할 것:

- `Server`, `Service`, `Connector`, `Engine`, `Host`, `Context`
- `server.xml`, `context.xml`, `web.xml`
- WAR 배포 구조
- 스레드 풀과 요청 처리 방식
- 세션과 `JSESSIONID`

여기서 "Context 하나가 웹 애플리케이션 하나"라는 감각을 잡으면 많이 정리된다.

## 5. 실험 위주로 공부하기

- 필터 하나 추가해서 요청 전후 로그 찍기
- 인터셉터 추가하기
- 에러 페이지 직접 매핑하기
- 세션 생성/만료 확인하기
- 정적 리소스가 톰캣에서 어떻게 서빙되는지 보기
- 동일 앱을 톰캣 재배포하면서 클래스 로딩 변화 보기

## 6. 로그를 읽는 습관 들이기

외장 톰캣에서는 로그가 공부 재료다.

특히 볼 것:

- 톰캣 시작 로그
- 스프링 컨텍스트 초기화 로그
- 서블릿 등록 로그
- 에러 발생 시 stack trace와 container log 차이

## 7. 내장 톰캣과 외장 톰캣 비교하기

같은 앱을 두 방식으로 각각 실행해 보면서 아래를 정리한다.

- 배포 방식 차이
- 설정 위치 차이
- 운영 관점 차이
- 디버깅 포인트 차이

## 추천 학습 루트

- 1주차: 서블릿, 톰캣, 스프링 MVC 요청 흐름 이해
- 2주차: WAR 배포, 외장 톰캣 설정, 로그 분석
- 3주차: 필터, 인터셉터, 세션, 에러 처리 실험
- 4주차: 내장 톰캣 vs 외장 톰캣 비교, 실제 배포 문서 정리

가장 중요한 건 "이론 먼저"보다 `작은 앱을 배포하고 요청 하나가 어디를 통과하는지 끝까지 추적`하는 방식이다. 이걸 3~4번 반복하면 스프링과 톰캣의 경계가 선명해진다.
